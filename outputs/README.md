# Outputs

This folder contains results generated by the pipeline.
The subfolders are created automatically when you run the scripts.

```
outputs/
├── maps/           ← PNG heatmaps and difference maps
│   ├── heatmap_a_car_only.png
│   ├── heatmap_b_car_transit.png
│   ├── heatmap_c_car_microtrans.png
│   ├── heatmap_d_car_for_all.png
│   ├── heatmap_e_transit_only.png
│   ├── heatmap_all5_scenarios.png      ← combined 5-panel
│   ├── heatmap_absolute_all5.png       ← absolute A_i values
│   ├── heatmap_difference_maps.png     ← 3 diverging difference maps
│   └── heatmap_slide_figure.png        ← slide-ready 3-panel
│
├── plots/          ← step plots and other charts
│   └── accessibility_step_plot.png     ← 5-scenario step plot
│
└── tables/         ← CSV results
    ├── accessibility_results.csv       ← Ã by scenario and P
    └── selected_stops_P4.csv           ← optimal P=4 stop locations
```

## Adding results to git

The output folders are in `.gitignore` by default (maps are large binary files).
To track specific results, add explicit exceptions in `.gitignore`:

```
# Track specific output files
!outputs/tables/accessibility_results.csv
!outputs/tables/selected_stops_P4.csv
!outputs/plots/accessibility_step_plot.png
```

Or commit the outputs folder entirely if you want a self-contained snapshot
for a conference submission.
